1. Type Hints

Type hints ka matlab hai ke hum Python ko batate hain ke variable ya function kis type ka data use karega.

Jaise:

int
str
float
bool

Is se programmer ko foran pata chal jata hai ke function kis type ka input expect kar raha hai aur kya return karega.

2. Generics (TypeVar)

TypeVar ka use tab hota hai jab ek hi function multiple data types ke sath kaam kar sakta ho.

Misal ke taur par ek function integer list aur string list dono ke liye kaam kare.

Is se:

Code dobara likhne ki zarurat nahi hoti.
Function reusable ban jata hai.
Type safety bhi maintain rehti hai.
3. Union (|)

Union ka matlab hai ke ek variable ya parameter ek se zyada data types accept kar sakta hai.

Misal:

int | float

Yani function integer bhi le sakta hai aur float bhi.

Is se function flexible ban jata hai.

4. Optional

Optional ka matlab hai ke value ho bhi sakti hai aur nahi bhi.

Python mein iska matlab hota hai:

str | None

Yani agar user name provide kare to string milegi, warna None bhi acceptable hai.

Ye missing values ko safely handle karne ke liye use hota hai.

5. Callable

Callable ka matlab hai ke ek function ko doosre function ke argument ke taur par pass karna.

Yani function data ke bajaye kisi aur function ko receive kare aur usay execute kare.

Ye callbacks aur reusable operations ke liye use hota hai.

6. Utility Modules

Utility modules wo Python files hoti hain jin mein reusable helper functions rakhe jate hain.

Misal:

math_utils.py
string_utils.py

Har module ka apna specific purpose hota hai aur un mein type hints use kiye jate hain taake code organized aur maintainable rahe.

For Task 2:

Enforce strict type checking across their local environment using mypy configured on strict mode.

These are the important concepts used:

1. Static Type Checking ⭐⭐⭐⭐⭐
Checking type errors before running the program.
Performed by mypy.
2. mypy ⭐⭐⭐⭐⭐
A static type checker for Python.
Reads type hints and detects type-related errors.

Example:

pip install mypy
3. Strict Mode ⭐⭐⭐⭐⭐
Enables the strictest type checking rules.

Example (mypy.ini):

[mypy]
strict = True
4. Type Hints ⭐⭐⭐⭐⭐
Specify types for:
Variables
Function parameters
Return values

Example:

def add(a: int, b: int) -> int:
    return a + b
5. Function Return Type Annotation ⭐⭐⭐⭐☆

Every function should specify what it returns.

Example:

def greet(name: str) -> str:
    return f"Hello {name}"
6. Parameter Type Annotation ⭐⭐⭐⭐☆

Every function parameter should have a type.

Example:

def divide(a: float, b: float):
7. Optional ⭐⭐⭐⭐☆

When a function may return None.

Example:

from typing import Optional

def divide(a: float, b: float) -> Optional[float]:
8. None Type ⭐⭐⭐☆☆

Understanding when a function can return None.

Example:

if b == 0:
    return None
9. Main Function ⭐⭐⭐☆☆

Organizing code using a main() function.

Example:

def main() -> None:
10. if __name__ == "__main__" ⭐⭐⭐☆☆

Ensures the program runs only when executed directly.

Example:

if __name__ == "__main__":
    main()
11. Type Safety ⭐⭐⭐⭐⭐

Making sure values match their declared types.

Example:

add(10, 20)      # Correct
add("10", 20)    # Type error
12. Running mypy ⭐⭐⭐⭐⭐

Checking a file:

mypy --strict example.py

Checking an entire project:

mypy --strict .
13. Type Errors ⭐⭐⭐⭐☆

Examples of errors mypy detects:

Wrong argument types
Missing return type annotations
Returning the wrong type
Incompatible assignments
Missing type hints (in strict mode)

For Task 3, these are the important concepts used in the code:

Concepts Used
1. Refactoring
Improving the internal structure of code without changing its behavior.
Converting raw dictionary returns into a better data structure.
2. Raw Dictionary
Returning data using a normal Python dictionary.

Example:

{
    "name": "Fahad",
    "age": 22,
    "cgpa": 3.85
}
Problem: Relies on string keys.
3. Stringly-Typed Access Bugs
Errors caused by misspelling dictionary keys.

Example:

student["nmae"]

instead of:

student["name"]
Results in a KeyError.
4. Typed Structures
Data structures with predefined fields and types.
Examples:
@dataclass ✅ (used in your code)
TypedDict
NamedTuple
Custom classes
5. Dataclass
Creates classes with less boilerplate code.
Automatically generates methods like __init__() and __repr__().
Stores structured data in a clean and readable way.
6. Type Hints
Defining the type of each attribute and function return value.

Example:

name: str
age: int
cgpa: float
7. Function Return Type Annotation
Specifying what a function returns.

Example:

def get_student() -> Student:
    ...
8. Object Access vs Dictionary Access

Dictionary Access

student["name"]

Object Access

student.name

Object access is cleaner, safer, and supported by IDE autocomplete.

9. Type Safety
Ensures values match their expected types.
Helps tools like mypy detect mistakes before runtime.
10. IDE Autocompletion
When using a typed structure, IDEs can suggest:
student.name
student.age
student.cgpa

This reduces typing mistakes.
